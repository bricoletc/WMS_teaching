Nextflow¶

Is a strong alternative workflow manager to snakemake, also with a lot of examples and users in bioinformatics.

The key difference is that Nextflow does not compute the workflow DAG before running: instead of pre-computing all dependencies like snakemake, it runs as soon as it sees a process which can consume data it has at its disposal.

To pass data around it relies on a more general concept of channels instead of files. Channels can be created inside and outside processes (the equivalent of snakemake rules) and can be manipulated in many ways in the workflow script.
